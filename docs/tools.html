All Tools Reference

📰 Blog

Forge Agent provides a powerful set of 37+ tools that the AI can use to interact with your system. Tools are grouped by category below.

File Operations

read_file

Read the full contents of a file. Optionally read specific line ranges.

ParameterTypeRequiredDescription
pathstring✅Path to the file
start_linenumberFirst line to read (1-indexed)
end_linenumberLast line to read (inclusive)
 
{
  "name": "read_file",
  "args": { "path": "src/index.js", "start_line": 1, "end_line": 50 }
}

write_file

Write (create or overwrite) a file with given content. Creates parent directories automatically.

ParameterTypeRequiredDescription
pathstring✅Destination file path
contentstring✅Full file content to write
 
{
  "name": "write_file",
  "args": { "path": "test.txt", "content": "Hello World" }
}

append_to_file

Append text to the end of an existing file (or create it if missing).

ParameterTypeRequiredDescription
pathstring✅File path
contentstring✅Text to append

replace_in_file

Find and replace text in a file. Supports regex patterns.

delete_file

Permanently delete a file.

move_file

Move or rename a file or directory.

copy_file

Copy a file to a new location.

get_file_info

Get metadata about a file or directory (size, modified date, line count, etc.).

list_directory

List files and folders in a directory, optionally recursive.

create_directory

Create a directory (and all necessary parent directories).

write_files

Write multiple files at once — useful for scaffolding projects.

Search & Navigation

search_in_files

Search for text patterns inside files (like grep -r). Returns matching lines with filenames.

search_codebase

Intelligent multi-file codebase search. Finds functions, classes, variables, and text by name or meaning.

find_files

Search for files by name pattern (glob-style, e.g. "*.js").

Shell & Processes

run_command

Execute a shell command and return its output. Runs in the working directory by default.

ParameterTypeRequiredDescription
commandstring✅Shell command to run
cwdstringWorking directory for the command
timeoutnumberTimeout in milliseconds
 
{
  "name": "run_command",
  "args": { "command": "npm test" }
}

start_process

Start a long-running background process (dev server, watcher, build tool).

stop_process

Stop a background process started with start_process.

list_processes

List all background processes started in this session.

read_process_logs

Read recent output from a background process.

Git Integration

git_status

Show the working tree status — staged, unstaged, and untracked files.

git_log

Show recent commit history.

git_diff

Show changes between commits, working tree and index, or two branches.

git_branches

List all local (and optionally remote) branches.

git_show

Show the full details of a specific commit.

git_blame

Show who last modified each line of a file.

Development

run_tests

Run the project's test suite and return structured pass/fail results.

install_package

Install one or more packages using the project's package manager.

diff_files

Generate a unified diff showing the differences between two files.

patch_file

Apply a unified diff patch to a file.

Environment

read_env

Read variables from a .env file. Secret values are automatically masked.

set_env_var

Set or update a single environment variable in a .env file.

delete_env_var

Remove a variable from a .env file.

list_env_files

Find all .env files in the project and list their variable names.

check_env_vars

Verify that all required environment variables are present.

System

take_screenshot

Capture a screenshot of the entire screen.

read_clipboard

Read the current text content from the system clipboard.

write_clipboard

Write text to the system clipboard.
