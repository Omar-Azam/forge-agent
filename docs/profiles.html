Agent Profiles

What are Profiles?

Profiles are pre-configured sets of instructions, tool preferences, and constraints that tune the agent for specific types of work. Using a profile helps the AI produce better code by focusing on the right patterns and tools for the job.

Available Profiles

default

General purpose configuration with no specific specialization. Balanced for a wide variety of tasks.

  • Max Iterations: 40
  • Planning Mode: Off

backend

Optimised for Node.js, Python, Go, and REST API development. Focuses on modularity, error handling, and security.

  • Max Iterations: 50
  • Planning Mode: On
  • Best for: API design, database migrations, server logic, auth systems.
forge-agent --profile=backend "add JWT auth to the API"

frontend

Optimised for React, Vue, HTML/CSS, and UI development. Focuses on component architecture, responsiveness, and accessibility.

  • Max Iterations: 40
  • Planning Mode: Off
  • Best for: Creating UI components, styling, client-side routing, state management.
forge-agent --profile=frontend "create a responsive navigation bar"

data-science

Optimised for Python data analysis, ML, and Jupyter notebooks. Focuses on pandas, numpy, and reproducible code.

  • Max Iterations: 35
  • Planning Mode: Off

devops

Optimised for Docker, CI/CD, shell scripts, and infrastructure. Focuses on idempotent operations and minimal images.

  • Max Iterations: 30
  • Planning Mode: On

Usage

Activate a profile using the --profile flag:

forge-agent --profile=backend "refactor the database layer"

To see all available profiles in your terminal:

forge-agent --list-profiles

Custom Profiles

You can create your own profiles by defining a JSON file and loading it with --custom-profile.

{
  "name": "my-custom-profile",
  "description": "Optimised for my internal projects",
  "systemPromptAddition": "Follow our internal style guide...",
  "maxIterations": 60,
  "planningMode": true
}
forge-agent --custom-profile=./profile.json "my task"
