🚀 Performance Benchmarks

Forge Agent includes a built-in benchmarking suite to measure the performance and reliability of its core logic layers. This helps ensure that new features don't introduce performance regressions.

Note: These benchmarks measure the agent's internal logic (parsing, tools, search) in isolation. They do not include AI response latency or browser automation time, as those depend on external factors like network speed and AI provider load.

What the Benchmarks Measure

The suite is divided into five main categories:

Category Measures
Parser JSON extraction, thinking block removal, and recovery from malformed output.
Tools Execution time of core tools like read_file, write_file, and run_command.
Search Fuzzy matching speed, symbol extraction (AST-like parsing), and codebase-wide search efficiency.
Truncation Speed of the smart truncation system when handling large code blocks or test outputs.
Memory Retrieval and formatting of project memory and task history.

Running Benchmarks

You can run benchmarks directly from the CLI:

# Run all benchmarks
forge-agent --benchmark

# Run only a specific category
forge-agent --benchmark=parser

# Run with more iterations for higher accuracy
forge-agent --benchmark --benchmark-iterations=10

Interpreting Results

Benchmarks are compared against a baseline (expected maximum time in ms).

  • ✅ PASS: Mean time is within the baseline.
  • ⚠️ WARN: Mean time exceeded baseline but is within 1.5x.
  • ❌ FAIL: Mean time exceeded 1.5x of the baseline.

Comparing Reports

You can compare your current performance against the previous run:

forge-agent --benchmark-compare

Using in CI/CD

Output results as JSON for automated performance tracking in GitHub Actions:

# In your CI workflow
- name: Run performance benchmarks
  run: forge-agent --benchmark-json > results.json

- name: Upload benchmark results
  uses: actions/upload-artifact@v4
  with:
    name: performance-report
    path: results.json

Contributing Benchmarks

New benchmarks can be added to src/benchmarks/. Each file must export a registration function:

module.exports = function register(suite) {
  suite.add('my new benchmark', async () => {
    // code to measure
  }, {
    category: 'tools',
    baseline: 50 // ms
  });
};
