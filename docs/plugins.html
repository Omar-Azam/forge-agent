Custom Plugins

What are Plugins?

Plugins allow you to add new tools to Forge Agent without editing the core source code. Any .js file placed in the plugins directory is automatically loaded and made available to the AI on every run.

Plugin Directory

By default, Forge Agent looks for plugins in:

~/.deepseek-agent/tools/

Writing Your First Plugin

A plugin is a standard Node.js module that exports an object with the following fields:

  • name: Unique tool name (lowercase, underscores only).
  • description: A clear explanation of what the tool does for the AI.
  • parameters: An object defining the arguments the tool accepts.
  • execute: An async function that performs the action and returns a string result.

Example: fetch_weather.js

module.exports = {
  name: 'fetch_weather',
  description: 'Get current weather for a city',
  parameters: {
    city: { type: 'string', required: true, description: 'City name' },
  },
  async execute({ city }) {
    const https = require('https');
    return new Promise((resolve, reject) => {
      https.get(`https://wttr.in/${city}?format=3`, res => {
        let data = '';
        res.on('data', c => data += c);
        res.on('end', () => resolve(data.trim()));
      }).on('error', reject);
    });
  },
};

Generating a Stub

Use the CLI to create a new plugin file with example boilerplate:

forge-agent --new-plugin my_tool

Disabling a Plugin

To temporarily disable a plugin without deleting the file, rename it to start with an underscore:

_my_tool.js

Plugin Constraints

  • Shadowing: Custom plugins cannot have the same name as built-in tools.
  • Output: The execute function must always return a string (the AI reads this string as the result).
  • Safety: Plugin errors are caught and shown as warnings; they will not crash the main agent loop.
