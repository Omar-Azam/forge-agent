Security Model

Security Overview

Forge Agent executes arbitrary commands and interacts with your local filesystem. To ensure your safety, we implement a multi-layered security model that handles trust boundaries between the AI and your machine.

Protected Paths

Certain system-critical and sensitive paths are always blocked from being read or written by the agent, regardless of configuration. These include:

  • System files: /etc/passwd, /etc/hosts, /etc/sudoers
  • Credentials: ~/.ssh, ~/.aws, ~/.npmrc, ~/.git-credentials
  • Configuration: ~/.gnupg, ~/.config/gh, ~/.docker/config.json

Secret Masking

Forge Agent automatically protects your secrets:

  • .env files: Automatically identified and masked. Values are replaced with **** (masked) before being sent to the AI.
  • Sensitive extensions: Files like .pem, .key, and .pfx trigger a security warning in the agent loop.
  • Output Redaction: Common secret patterns (API keys, JWTs, long hex strings) are redacted from the tool outputs and history logs.

Sandbox Modes

You can control the level of filesystem access the agent has:

  • Standard Mode (Default): Blocks sensitive paths but allows access outside the working directory (e.g., to read global node_modules or system documentation).
  • Strict Sandbox (STRICT_SANDBOX: true): Blocks ALL file access outside the current working directory. It also performs symlink escape detection to prevent the agent from following symlinks to outside locations.

Command Safety

Dangerous commands that could cause irreversible damage are blocked by default:

  • Destructive: rm -rf /, rm -rf ~, mkfs
  • Exploits: :(){ :|:& };: (fork bomb)

To allow these commands for advanced use cases, set ALLOW_DANGEROUS_COMMANDS: true.

Audit Logging

Enable AUDIT_LOG: true to record every tool call the agent makes. This creates an append-only log file at ~/.deepseek-agent/audit.log in JSONL format, which you can use for security reviews.

Reporting Vulnerabilities

Please report security issues privately via GitHub Security Advisories or as specified in our SECURITY.md file.
